# Sprint-19: Copilot 与查询资产中心协同工作流

**时间**: 2026-03
**状态**: READY
**目标**: 为 `查询` 与 `AI Copilot` 建立统一的分析草稿工作流，打通“对话 -> 草稿 -> 正式查询 -> 可视化”的资产晋升链。

## 背景

当前 `Copilot` 结果更多停留在聊天消息和 URL 级 SQL 跳转，`查询` 页面又主要管理正式 Query/Card。  
缺少中间层会导致：

- Copilot 分析结果难以沉淀
- 正式查询目录容易被试验性 SQL 污染
- 查询、可视化、仪表盘、大屏之间衔接不顺
