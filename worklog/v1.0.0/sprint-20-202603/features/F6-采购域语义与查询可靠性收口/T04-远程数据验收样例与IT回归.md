# T04: 远程数据验收样例与 IT 回归

**优先级**: P1
**状态**: DONE
**依赖**: T02, T03

## 目标

把采购域修复建立在真实远程数据之上，而不是只在本地 schema 上做静态推断。

## 技术设计

- 以 `ai.yuzhicloud.com` 绑定的 `rs_cloud_flower` 最新库作为验收基线
- 固化本次核心样例：
  - 问句：`查询2025年2月，绿萝这个产品的采购详细情况，按采购人、采购金额统计`
  - 精确口径：`good_name='绿萝'`
  - 结果基线：
    - 明细行数 `158`
    - 总数量 `191`
    - 总金额 `21534.50`
    - `邹顿顿` `11979.50`
    - `王果` `9555.00`
- 区分精确匹配与模糊 `%绿萝%` 的结果差异，防止验收口径漂移
- 补齐 IT 验收文档，并提供可执行自动化 smoke
