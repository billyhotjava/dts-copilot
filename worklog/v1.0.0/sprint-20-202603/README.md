# Sprint-20: 分析工作台双入口产品化

**时间**: 2026-03
**状态**: IN_PROGRESS
**目标**: 把 `查询` 与 `AI Copilot` 的平级关系产品化，形成统一的分析工作台与来源追溯链。

## 背景

`sprint-19` 已完成 `analysis_draft` 基础工作流，但整体体验仍偏“功能打通”。  
下一阶段需要把它提升成真正的产品层能力：

- 统一来源上下文
- 强化查询资产中心
- 打通 Copilot 与菜单页的往返
- 让多端资产保留来源追溯

此外，前几轮主题域治理已经沉淀了 `project / field-operations / ELT` 基线，但采购域仍缺少可落地的语义资产与 authority SQL。  
本 sprint 追加一条采购域收口工作流，用来把“分析工作台产品化”与“高价值业务问句可用性”接起来。
