# CS-10: 数据源表单收敛与错误透传

**状态**: DONE
**依赖**: CS-06

## 目标

修复手动添加数据源时的弱约束问题，避免数据库类型自由输入、空字段直达后端，以及上游 AI 校验失败被吞成泛化 502。

## 技术设计

- 手动添加页把数据库类型改为下拉框
- 当前仅保留表单已支持的类型：
  - PostgreSQL
  - MySQL
- 提交前做必填校验：
  - 数据源名称
  - 数据库类型
  - 主机
  - 数据库名
- `analytics` 创建数据源时保留 `copilot-ai` 的真实错误信息

## 完成标准

- [ ] 数据库类型不再是自由文本输入
- [ ] 手动添加页缺少 `host` 或 `database` 时不再发请求
- [ ] 用户能直接看到真实后端错误，而不是“创建数据源失败”
- [ ] 相关前后端回归测试通过
