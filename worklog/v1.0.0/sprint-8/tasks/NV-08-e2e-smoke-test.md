# NV-08: 端到端冒烟测试与评测用例

**状态**: READY
**依赖**: NV-01~06

## 目标

验证完整的 NL2SQL → 可视化链路，创建评测基线。

## 测试计划

### 冒烟测试场景（5-10 个）

配置一个真实业务数据库（或测试库），依次验证：

| # | 自然语言问题 | 预期 SQL 类型 | 验证点 |
|---|-------------|-------------|--------|
| 1 | "查看所有项目列表" | 单表 SELECT | schema_lookup 召回正确表 |
| 2 | "本月新增项目数量" | 单表 COUNT + 日期过滤 | 聚合和日期条件 |
| 3 | "各部门费用合计" | 单表 GROUP BY | 分组聚合 |
| 4 | "费用最高的前10个项目" | 单表 ORDER BY + LIMIT | TOP-N |
| 5 | "各项目的养护次数和费用" | 2 表 JOIN | 简单 JOIN |

### 验证清单（每个场景）

- [ ] CopilotChat 选择数据源后发送问题
- [ ] Agent 调用 schema_lookup（日志可见）
- [ ] Agent 回复包含 SQL 代码块
- [ ] SQL 通过安全校验（只读、无危险关键字）
- [ ] "创建可视化" 按钮出现
- [ ] 点击后跳转 CardEditorPage
- [ ] SQL 自动执行，结果正确渲染
- [ ] 图表类型可切换

### 评测基线

- 通过 `Nl2SqlEvalPage` 创建 10-20 个评测用例
- 覆盖：单表查询、JOIN、聚合/分组、日期过滤、TOP-N
- 执行评测，记录基线 pass rate 和 score
- 目标：pass rate ≥ 60%

### 失败分析

对失败的 case 分类记录：
- **召回不准**：schema_lookup 返回了错误的表 → 需补充同义词（NV-07）
- **SQL 生成错误**：LLM 生成了语法错误或逻辑错误的 SQL → 需优化提示词（NV-03）
- **执行失败**：SQL 正确但执行超时/权限不足 → 检查数据源配置

## 影响文件
