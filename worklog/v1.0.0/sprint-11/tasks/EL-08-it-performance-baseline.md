# EL-08: IT 集成测试与性能基准

**优先级**: P2
**状态**: READY
**依赖**: EL-01~07

## 目标

验证主题层的正确性和性能，建立查询响应时间基线。

## 测试维度

### 1. 同步正确性

| 编号 | 场景 | 验证方式 |
|------|------|---------|
| ES-01 | 首次全量同步 | mart 表行数 = 活跃项目数 × 同步天数 |
| ES-02 | 增量同步幂等 | 重复执行不产生重复数据 |
| ES-03 | watermark 推进 | 每次同步后 watermark 正确更新 |
| ES-04 | 状态码翻译 | fact 表中 biz_type_name 为中文 |
| ES-05 | 同步失败恢复 | 失败后下次同步从上次 watermark 继续 |

### 2. 查询性能基准

| 编号 | 场景 | 主题层目标 | 视图层对照 |
|------|------|-----------|-----------|
| EP-01 | 单项目 3 月绿植趋势 | < 500ms | < 3s |
| EP-02 | 全项目月度加花排行 | < 1s | < 5s |
| EP-03 | 半年事件类型分布 | < 500ms | < 5s |
| EP-04 | 项目经营综合排名 | < 1s | N/A（视图层无法直接完成） |

### 3. 路由正确性

| 编号 | 问句 | 期望层 | 期望表 |
|------|------|--------|--------|

| ER-04 | 待审批的报花单 | VIEW | v_flower_biz_detail |

### 4. 降级验证

| 编号 | 场景 | 期望行为 |
|------|------|---------|
| ED-01 | mart 表为空 | 自动回退到视图层 |
