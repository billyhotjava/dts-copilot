# MD-04 共享参考主数据模型

**优先级**: P1  
**状态**: DONE

## 目标

明确库房、供应商、组织、角色、字典等共享参考主数据的治理范围和优先级。

## 范围

- 库房
- 供应商
- 员工 / 组织 / 角色
- 状态字典
- 来源码 / 结算码 / 付款码

## 共享参考对象

### 已有明确对象

- 库房
- 员工 / 角色 / 组织
- 字典与状态枚举

### 当前缺口对象

- 供应商
- 付款 / 结算 / 来源码的统一字典

## 重点结论

### 库房

库房已经是比较清晰的参考主数据，应保留为独立主数据对象，不要混在库存结果面里。

### 供应商

供应商是当前最大的参考主数据缺口。

现状：

- 前端存在供应商页面
- 采购和合同会用到供应商
- 后端未见清晰独立供应商控制器

结论：

供应商必须在后续实现阶段升级为独立主数据，否则：

- 采购报表难以稳定按供应商聚合
- 应付、采购对账、供应商排行口径不稳定

### 字典与状态

当前大量状态值直接写在页面里，例如：

- 项目状态
- 库存状态
- 凭证来源
- 月账单状态

这类值必须从“页面硬编码”提升为参考主数据或统一字典，不然固定报表和 Copilot 都无法稳定消费。

## 优先级建议

1. 库房
2. 供应商
3. 组织 / 角色
4. 状态与来源字典
5. 财务参考字典
