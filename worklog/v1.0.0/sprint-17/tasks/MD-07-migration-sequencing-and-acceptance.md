# MD-07 迁移顺序与验收基线

**优先级**: P1  
**状态**: DONE

## 目标

明确主数据治理的渐进式实施顺序和阶段性验收口径。

## 阶段建议

1. 项目轴主数据
2. 物品轴主数据
3. 共享参考主数据
4. 固定报表 / Copilot 主数据接入

## 分阶段路线图

### 阶段 1：项目轴

- 客户、合同、项目点、摆位对象树固定
- 项目报表、任务、报花、结算先统一挂项目点

### 阶段 2：物品轴

- 物品、分类、属性、规格、价格、别名统一
- 采购、库存、报花、实摆先统一挂物品轴

### 阶段 3：共享参考主数据

- 库房、供应商、组织、字典统一
- 固定报表和财务采购类模板优先消费这些对象

### 阶段 4：固定报表 / Copilot 接入

- 固定报表目录全面切到主数据受控取数面
- Copilot 的固定报表命中和探索式 SQL 都基于主数据轴工作

## 验收标准

1. 页面/API/数据库归属唯一  
2. 项目和物品具备稳定业务键  
3. 供应商和库房不再只是页面附属信息  
4. 状态码、来源码具备统一来源  
5. 固定报表和 Copilot 可明确说明自己依赖的主数据对象

## 风险与回退

- 不做表结构大爆炸改造
- 优先做语义统一、归属统一、消费约束统一
- 必要时允许旧页面继续使用旧表，但新报表和新 Copilot 路径必须走受控主数据面
