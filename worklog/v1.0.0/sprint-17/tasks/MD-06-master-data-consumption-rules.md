# MD-06 主数据消费规则

**优先级**: P1  
**状态**: DONE

## 目标

为固定报表和 Copilot 建立统一的主数据消费规则，避免继续直接依赖混乱业务表和页面硬编码。

## 范围

- 固定报表模板
- Copilot 模板优先路由
- 受控取数面
- 页面状态码/来源码治理

## 固定报表消费规则

1. 所有固定报表必须至少挂一个主数据轴  
2. 聚合报表必须显式声明按哪个主数据对象聚合  
3. 页面级状态文案不能再作为报表唯一口径来源  
4. 供应商、库房、项目、物品必须可回挂到主数据对象

## Copilot 消费规则

1. 先识别问题中的主数据对象  
2. 再命中固定报表模板  
3. 未命中才进入探索式 SQL  
4. 探索时只允许在受控取数面里使用主数据键和事实表关系

## 新页面/新接口编码约束

1. 不允许在新页面中再手写新的状态语义  
2. 不允许交易页面重新定义物品和项目名称口径  
3. 不允许以库存余额页、汇总页结果反向充当主数据  
4. 固定报表模板必须记录其依赖的主数据轴
