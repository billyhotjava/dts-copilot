# MD-05 主数据与交易事实边界表

**优先级**: P0  
**状态**: DONE

## 目标

把当前系统中的主数据、配置型事实、交易事实和派生事实重新划边界，避免后续治理对象失焦。

## 重点对象

- 主数据：customer / contract / project / goods / storehouse / supplier
- 配置型事实：project_green / contract_rent_fee / settlement_config
- 交易事实：purchase / stock movement / flower_biz / task / month_account / voucher

## 边界矩阵

### 主数据

- 客户
- 合同
- 项目点
- 摆位
- 物品
- 分类 / 属性 / 规格 / 价格
- 库房
- 供应商
- 组织 / 角色 / 字典

### 配置型事实

- 项目绿植
- 合同租金规则
- 项目结算配置
- 项目额外绿植配置

### 交易事实

- 采购单
- 配送单
- 入库单 / 出库单
- 调拨 / 退货 / 报损
- 报花单
- 任务项
- 月账单 / 对账单
- 付款 / 收款 / 凭证

### 派生事实 / 结果面

- 库存余额
- 采购汇总
- 月度汇总
- 经营排行
- Dashboard / Screen 卡片结果

## 风险清单

1. 把库存余额当库存主数据  
2. 把月账单当财务主数据  
3. 把项目绿植和合同租金规则当项目主数据本体  
4. 把采购汇总和报表结果反向当作采购事实来源

## 约束建议

- 固定报表优先消费主数据 + 事实表，不直接依赖页面汇总结果
- Copilot 优先识别主数据对象，再决定进入模板或探索
