# MD-03 物品轴 Canonical Model

**优先级**: P0  
**状态**: DONE

## 目标

定义物品主数据的标准对象树、命名规范、识别键和价格/别名规则。

## 范围

- 物品
- 分类
- 属性
- 规格
- 价格
- 项目别名

## Canonical Model

### 对象树

1. `Goods`
2. `GoodsClassify`
3. `GoodsAttribute`
4. `GoodsAttributeItem`
5. `GoodsPrice`
6. `GoodsAlias`

### 稳定识别建议

对外应优先使用：

- 物品编码
- 标准物品名
- 规格组合
- 价格版本

而不是仅依赖当前页面上的自由文本名称。

### 别名策略

当前项目侧已存在 `ProjectGoodAlias`，说明真实业务存在：

- 标准物品名
- 项目内部叫法
- 采购/仓库/报花的不同叫法

建议后续统一为：

- `standard_name`
- `alias_type`
- `alias_scope`（全局 / 项目）
- `alias_value`

而不是把别名逻辑散落在项目页、采购页和报花页里。

### 价格策略

`GoodsPrice` 应作为物品主数据的从属定义，而不是库存或采购页面可随意重定义的事实。

当前风险点：

- 库存页存在修改价格能力
- 报花、采购、库存、销售各处都有价格语义

建议：

- 价格以“标准价格 + 使用场景价”分层
- 页面只能消费，不应各自定义主价格口径

## 与业务域的关系

- 采购：消费物品标准定义与价格
- 库存：消费物品与条码/SKU
- 报花：消费物品标准定义与项目别名
- 项目实摆：消费物品标准定义与摆位映射
