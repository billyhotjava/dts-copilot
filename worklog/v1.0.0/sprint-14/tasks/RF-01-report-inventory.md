**目标**: 建立首批已知报表库存，为后续固定报表模板、目录化入口和 Copilot 探索回灌提供统一基线。

## 完成检查

- [x] 至少 20 个候选报表
- [x] 每条记录包含 `domain`、`user role`、`freshness`、`display type`、`owner`
- [x] 财务与采购相关报表均标记为 `P0`

## Inventory Schema

| 字段 | 含义 |
|------|------|
| `report_id` | 候选报表编号 |
| `report_name` | 报表名称 |
| `domain` | 所属业务域 |
| `user_role` | 主要使用角色 |
| `freshness` | 数据新鲜度要求 |
| `display_type` | 展示形式 |
| `owner` | 业务口径负责人 |
| `priority` | `P0` / `P1` / `P2` |
| `status` | 候选 / 待确认 / 已固化 |
