# T01: 财务域典型问句回归集

**优先级**: P0
**状态**: READY
**依赖**: F1 / F2 完成

## 目标

把财务域 8+ 条典型问句固化为可重复回归的基线集，每条记录"问句 → 期望路由 → 期望视图 → 期望 SQL 关键特征 → 期望结果摘要"。

## 技术设计

### 1. 回归集结构

回归文档 `it/finance-query-regression.md`，每条记录如下：

```
## R1: 客户欠款排行前 10

- 问句：客户欠款前 10 是谁？
- 期望路由：domain=finance, template=FIN-Q-CUSTOMER-AR-RANK-TOPN
- 期望视图：mart.finance.customer_ar_rank_daily
- SQL 关键特征：
  - 必有 `WHERE snapshot_date = (SELECT MAX(snapshot_date) ...)`
  - 必有 `ORDER BY outstanding_amount DESC LIMIT 10`
  - 不应出现 `GROUP BY customer_name`（mart 已聚合）
- 期望结果（远程 ai.yuzhicloud.com）：
  - 行数：≤ 10
  - 第一行欠款金额：≥ 第二行欠款金额
  - 列名包含：欠款金额, 30天内, 30到60天, 60到90天
- 反例（必须不出现）：
  - 走 settlement_summary 现场 SUM
  - 漏掉 snapshot_date 过滤
```

### 2. 回归集清单（首批 12 条）

按 sprint README 8 条样本 + 4 条边缘 case：

| ID | 问句 | 期望模板 | 关键校验点 |
|---|---|---|---|
| R1 | 客户欠款前 10 是谁？ | `FIN-Q-CUSTOMER-AR-RANK-TOPN` | 取最新 snapshot_date |
| R2 | 本月各项目应收多少？ | `FIN-Q-MONTHLY-SETTLEMENT-BY-PROJECT` | account_period 用 `DATE_FORMAT(CURDATE(),'%Y-%m')` |
| R3 | 2025 年 3 月哪些项目还没收款完？ | `FIN-Q-COLLECTION-PROGRESS-BY-PROJECT` | account_period='2025-03' AND collection_rate < 1.0 |
| R4 | 万象城那个项目 2025 年回款进度 | `FIN-Q-COLLECTION-PROGRESS-BY-PROJECT` | project_name LIKE '%万象城%' AND account_period LIKE '2025-%' |
| R5 | 待付款审批里有没有超过 30 天的？ | `FIN-Q-PENDING-PAYMENT-OVERDUE` | DATEDIFF(CURDATE(), submit_time) > 30 |
| R6 | 上月预支单还有多少没核销？ | `FIN-Q-ADVANCE-OUTSTANDING` | outstanding_advance > 0，时间窗口=上月 |
| R7 | 2025 年 Q1 总开票金额 | `FIN-Q-INVOICE-AMOUNT-PERIOD` | invoice_date BETWEEN '2025-01-01' AND '2025-04-01' |
| R8 | 张三的报销单到哪一步了？ | `FIN-Q-REIMBURSEMENT-BY-APPLICANT` | applicant_name='张三' |
| R9 | 客户 A 还欠多少？ | `FIN-Q-CUSTOMER-AR-DETAIL` | customer_name LIKE '%客户A%' |
| R10 | 应收账款情况（边缘 / 模糊） | `FIN-Q-RECEIVABLE-OVERVIEW-KPI` | as_of_date 默认今日 |
| R11 | 项目租金（边界 / 路由分流） | 走 project 域 `v_monthly_settlement`，**不**走 finance | 验证不被 finance 抢路由 |
| R12 | 采购金额前 10（域冲突测试） | 走 procurement 域，**不**被 finance 拦 | 验证 priority 设计有效 |
