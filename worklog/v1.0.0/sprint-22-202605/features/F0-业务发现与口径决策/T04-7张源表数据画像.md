## 目标

对 7 张报花源表跑数据画像，把"代码文档说的"和"生产实际是的"对齐，提前发现脏数据 / 状态漂移 / 时间断点 / 软外键孤儿等问题。F4-T02 stg 层的清洗规则、schema.yml 测试的 severity 全部基于本 task 输出。

## 标准画像 SQL

每张表跑下列检查（生产库 read-only）：

```sql
-- A. 主键唯一性
SELECT COUNT(*), COUNT(DISTINCT id) FROM <table> WHERE del_flag='0';

-- B. 表大小与时间分布
SELECT 
  COUNT(*) AS total,
  MIN(create_time) AS first_record,
  MAX(create_time) AS last_record,
  DATE_FORMAT(MIN(create_time), '%Y-%m') AS first_month,
  DATE_FORMAT(MAX(create_time), '%Y-%m') AS last_month
FROM <table> WHERE del_flag='0';

-- C. 时间分布（按月）—— 找断点
SELECT 
  DATE_FORMAT(create_time, '%Y-%m') AS m,
  COUNT(*) AS cnt
FROM <table> WHERE del_flag='0'
GROUP BY m ORDER BY m;

-- D. 各关键字段空值率
SELECT 
  COUNT(*) AS total,
  SUM(CASE WHEN <field> IS NULL OR <field> = '' OR <field> = 0 THEN 1 ELSE 0 END) AS null_count,
  ROUND(100.0 * SUM(CASE WHEN <field> IS NULL OR <field> = '' OR <field> = 0 THEN 1 ELSE 0 END) / COUNT(*), 2) AS null_pct
FROM <table> WHERE del_flag='0';

-- E. 枚举字段实际分布
SELECT <enum_field>, COUNT(*) AS cnt
FROM <table> WHERE del_flag='0'
GROUP BY <enum_field>
ORDER BY cnt DESC;

-- F. 异常值
SELECT COUNT(*) FROM <table>
WHERE <amount_field> < -1000000 OR <amount_field> > 100000000;

-- G. 软外键孤儿（针对 item 表）
SELECT COUNT(*) AS orphans
FROM t_flower_biz_item i
LEFT JOIN t_flower_biz_info f ON f.id = i.flower_biz_id
WHERE i.del_flag='0' AND f.id IS NULL;

-- H. 时间字段单调性（针对 incremental 模型考虑）
SELECT COUNT(*) FROM <table>
WHERE update_time < create_time;
```

## 重点关注的画像问题

### t_flower_biz_info

- **status 实际分布**：文档说 7 状态，生产是不是真的有这 7 个？还是大量数据是某 1-2 个状态？
- **biz_type 实际分布**：文档说 13 种，是不是有些 bizType 实际从未使用？
- **time 字段空值率**：`apply_time` / `start_lease_time` / `settlement_time` / `finish_time` 各自空值率，决定能不能作业务时间字段
- **历史数据时间分布**：哪一年开始有数据？是不是 2024 年之前数据极不完整？

### t_flower_biz_item

- **flower_biz_id 孤儿率**：与 `t_flower_biz_info` 的 LEFT JOIN 匹配率
- **status 与主表一致性**：item.status 是不是和 info.status 同步？还是各种漂移？
- **start_time / end_time 空值率**：决定起租字段能不能作业务时间

### t_flower_rent_time_log

- **变更频率**：起租时间被事后修改的实际比例
- **rent_time_type 实际值**：是不是真的 1/2 两种
