**优先级**: P0

## 目标

## 5 个决策

### 决策 1：报花的"业务时间"是什么？

**问题**：当业务方说"本月报花"时，"本月"对应哪个时间字段？

**候选**：

| 选项 | 字段 | 业务含义 | 推荐度 |
|---|---|---|---|
| A | `apply_time` | 申请时间 | 中 —— 反映业务发生时点，但跨月跨年长尾 |
| B | `start_lease_time` | 起租时间 | **高** —— 反映租金归属期间 |
| C | `settlement_time` | 结算时间 | 低 —— 滞后；NULL 多 |
| D | `finish_time` | 完工时间 | 中 —— 反映物理完成；与起租可能差几天 |
| E | `create_time` | 创建时间 | 低 —— 仅审计用，不应作业务时间 |

**访谈引导**：

```
问：当老板早会问"3 月份加摆了多少"，他想看的是：
  - A 3 月份提交的报花申请？
  - B 3 月份起租生效的绿植？
  - C 3 月份财务结算完的报花？

如果一张报花单：
  - 2025-02-28 申请提交
  - 2025-03-05 审核通过
  - 2025-03-10 配送到现场
  - 2025-03-15 起租
  - 2025-04-05 财务结算
那这张单算"哪个月"的报花？
```

**输出**：业务方书面拍板（含至少 1 个具体例子佐证）。

### 决策 2：13 bizType 在 mart 里怎么分组？

**候选**：

| 选项 | 分组方式 | mart 数量 | 推荐度 |
|---|---|---|---|
| A | 每种独立 | 13 张 ads | 低 —— 太碎 |
| B | 5 类合并：lease(1/2/3/4) / sale(7) / gift(8) / baddebt(6) / material(10/11/12) | **5-8 张 ads** | **高** |
| C | 不聚合，所有 bizType 混一张 | 1 张 ads | 低 —— LLM 易错聚合 |

**访谈引导**：

```
问：日常报表上，有没有同时看"加花 + 撤花 + 调花"汇总的需求？还是必须分开看？
问：销售（biz_type=7）的金额会和"加花租金"放在一起算吗？还是完全分开管理？
问：坏账（biz_type=6）属于"财务问题"还是"业务问题"？谁在看？
```

### 决策 3：异步联动的"数据滞后"如何处理？

**问题**：报花单 status=2 后，PlanPurchaseItem 异步生成；如果用户在 status=2 后 30 秒查"待入库的报花"，能查到吗？

**候选**：

| 选项 | 策略 | 影响 |
|---|---|---|
| A | mart 物化时强制 JOIN 上下游 | 数据准确但有滞后 |
| B | 分两个 mart：含未结算 + 仅已结算 | 查询者按需选；推荐 |
| C | 实时穿透 adminapi | 准确但慢且耦合 |

**访谈引导**：

```
问：业务方查"待付款审批的报花"时，可以接受看到的数据是几小时前的吗？
问：如果有"实时"和"次日"两个版本，业务方更愿意用哪个？
```

### 决策 4：金额符号在 mart 层规范化吗？

**问题**：bizType=2 的 `biz_total_rent` 是正数，bizType=3 是负数。mart 层保留原符号还是统一变正？

**候选**：

| 选项 | 处理 | 示例 |
|---|---|---|
| A | 保留原符号 | `SUM(biz_total_rent) = 1000 + (-300) = 700` |
| B | 统一变正 + amount_direction 字段 | `lease_in=1000, lease_out=300, net=700`（推荐） |
| C | 拆为 amount_in / amount_out 两列 | 同 B 但更直观 |

### 决策 5：销售/赠花是否进财务 mart？

**问题**：bizType=7（售花）和 bizType=8（赠花）走 ISaleAccountService，**不进** SettlementItem。是否在报花 mart 里做统一聚合？

**候选**：

| 选项 | 处理 | 影响 |
|---|---|---|
| A | 完全独立的 sale_summary mart | 推荐 —— 业务清晰 |
| B | 合并到 lease_summary | 容易聚合错 |
| C | 提供 UNION 视图 | A + 可选 UNION |

## 访谈方式

- 5 个决策一次性安排 1.5 小时业务方会议

- 主持：技术负责人
- 输出：录音 + 文字纪要
