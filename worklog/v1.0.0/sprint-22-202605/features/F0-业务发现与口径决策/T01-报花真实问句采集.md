# T01: 报花真实问句采集

**优先级**: P0
**状态**: READY
**依赖**: 无

## 目标

收集真实业务场景中关于"报花"的口语化问句，作为 F1 语义包 fewShots 与 F2 query template 的设计输入。**不要凭空设计问句**——业务方实际怎么问，模型就要怎么理解。

## 技术设计

### 1. 采集来源

| 来源 | 采集方式 | 关注点 |
|---|---|---|
| 钉钉 / 企业微信 / IM 群 | 翻最近 3 个月聊天记录，关键词搜 "报花 / 加摆 / 撤摆 / 换花 / 调花" | 项目群 / 财务群 / 早会群 |
| 邮件 | 搜邮件标题含报花相关词 | 老板对项目经理的批注 |
| 周会 / 月会纪要 | 翻会议记录 | "上月报花量"、"加摆 TOP 项目" 等 |
| 现场访谈 | 找 5 类角色各 1 人，问"你日常关于报花最常想知道什么？" | 养护人 / 项目经理 / 业务经理 / 财务 / 老板 |

### 2. 采集格式

每条问句必须**原样保留**（不改写、不润色），并标注：

```
## Q025
- 原话：我那个万象城上周加摆了几盆？
- 时间：2025-12-15 14:32
- 渠道：项目群
- 问者角色：养护人
- 关心 bizType：2 加花
- 关心粒度：项目 × 周
- 是否报表能直接答：能（list-add-flower.vue）
- 备注：上下文是核对结算前数量
```

### 3. 采集规模

| 角色 | 目标问句数 | 优先 bizType |
|---|---|---|
| 养护人 | 30 | 2/3/1 加摆/撤摆/换花 |
| 项目经理 | 30 | 全部，特别是 4 调拨 / 21 驳回 |
| 业务经理 | 20 | 7 售花 / 8 赠花 |
| 财务 | 15 | 6 坏账 / 4-5 状态结算 |
| 老板 | 10 | 月度趋势 / KPI |
| **合计** | **≥ 105** | 覆盖全 13 bizType |

### 4. 分类与归档

采集完成后按 4 个维度分组：

- **bizType 维度**：13 类 + "跨类"（如"本月所有报花"）
- **粒度维度**：项目 / 客户 / 养护人 / 摆位 / 全公司
- **时间维度**：实时 / 当日 / 本周 / 本月 / 本季 / 本年 / 自定义区间
- **意图类型**：列表（找具体单据）/ 汇总（数量金额）/ 状态查询 / 排行 / 对比 / 异常发现

每个维度生成统计图（条数分布），帮助 F2-T01 决定 query template 优先级。
